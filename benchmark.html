Key Insights

The "Identical Buttons" Problem (T4)

When multiple elements share the same label, DOM-based tools are fundamentally blind. You can pick by index, but you can't know what you're clicking. BeakVision reads the surrounding visual context — prices, plan names, card layouts — to make informed decisions. This is the core value proposition for AI agents on unknown sites.

Visual Verification is a Superpower (T5)

agent-browser can click a tab but cannot confirm the UI actually changed. Snapshots only show interactive elements — not labels, badges, colors, or state indicators. BeakVision screenshots + analysis provides complete visual verification, essential for reliable multi-step automation.

agent-browser Wins on Speed

For tasks where DOM refs suffice (T1, T2), agent-browser is ~2x faster. No screenshot overhead, no API calls. When you know the site and elements are well-labeled, it's the pragmatic choice.

keyboard.type() > fill()

The consistent T3/T5 fill failure reveals a real-world problem: modern React/Vue components often intercept input events. Vision-guided click + keyboard.type() bypasses framework abstractions entirely — the browser just sends keystrokes to whatever has focus.

<$0.01 for Full Visual Understanding

16 API calls, 25,941 tokens, under a penny. At ~1,600 tokens and ~3.3s per call, BeakVision adds modest overhead for dramatic capability gains. The cost-effectiveness makes it viable for production AI agents, not just demos.

The Best Approach Depends on the Task

Simple nav on known sites? agent-browser. Complex flows on unknown sites? BeakVision + Playwright. The hardest tasks — visual disambiguation, state verification, custom inputs — are only possible with vision.
